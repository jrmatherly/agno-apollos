All project operations use [mise](https://mise.jdx.dev) as the task runner. Tasks are file-based scripts in `mise-tasks/`.

```bash
# List all available tasks
mise tasks

# Run a task
mise run <task>
```

## Backend tasks

| Task | Description | Flags |
|------|-------------|-------|
| `mise run setup` | Install all dependencies (backend + frontend) | `--ci` for locked/frozen mode |
| `mise run format` | Format code (ruff format + import sorting) | `--check` for CI check-only |
| `mise run lint` | Lint code (ruff check) | |
| `mise run typecheck` | Type-check code (mypy) | |
| `mise run validate` | Run all checks (format-check + lint + typecheck) | |
| `mise run ci` | Full CI pipeline (install + validate) | |
| `mise run clean` | Clean build artifacts and caches | |

## Docker tasks

| Task | Description | Flags |
|------|-------------|-------|
| `mise run docker:up` | Start full stack | `--prod` for GHCR images, `--docs` to include docs |
| `mise run docker:down` | Stop all services | `--prod` for production compose |
| `mise run docker:logs` | Tail logs | `--prod` for production compose |
| `mise run docker:build` | Build images locally | `--platform amd64\|arm64` |
| `mise run dev` | Docker Compose watch (hot-reload) | |
| `mise run db` | Start database only | |

## Release workflow

```bash
mise run release
```

The release task:

1. Runs `mise run validate` to ensure code quality
2. Prompts for the new version number
3. Checks that CI is passing on the latest main commit
4. Bumps versions in `pyproject.toml`, `frontend/package.json`, and `uv.lock`
5. Creates a git tag
6. Creates a GitHub release via `gh release create`
7. Triggers the Docker image build workflow
