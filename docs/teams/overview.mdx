Teams combine multiple agents under a coordinator that delegates subtasks and synthesizes results.

## Coordinator Team

The Coordinator is the primary user-facing team. It analyzes each query and routes to the best specialist agent automatically.

```python
from agno.team import Team
from agno.team.team import TeamMode
from agno.tools.reasoning import ReasoningTools

from backend.agents.data_agent import data_agent
from backend.agents.knowledge_agent import knowledge_agent
from backend.agents.mcp_agent import mcp_agent
from backend.agents.reasoning_agent import reasoning_agent
from backend.agents.web_search_agent import web_search_agent
from backend.db import get_postgres_db
from backend.models import get_model

coordinator_team = Team(
    name="Apollos Coordinator",
    id="apollos-coordinator",
    mode=TeamMode.coordinate,
    model=get_model(),
    db=get_postgres_db(),
    members=_get_members,  # callable factory, defers M365 agent inclusion
    tools=[ReasoningTools(add_instructions=True)],
    instructions=[instructions],
    markdown=True,
    compress_tool_results=True,
    max_iterations=5,
    share_member_interactions=True,
    enable_agentic_memory=True,
    enable_session_summaries=True,
)
```

### Routing rules

| Query type | Routed to |
|-----------|-----------|
| Documents, policies, knowledge base, file browsing | Knowledge Agent |
| Connected tools, integrations, API docs | MCP Agent |
| SQL queries, data analysis, metrics, table lookups | Data Agent |
| Current events, external information, web lookups | Web Search Agent |
| Multi-step reasoning, comparisons, structured analysis | Reasoning Agent |
| Email, calendar, OneDrive, SharePoint, Teams | M365 Agent (when enabled) |

### Specialist hints

The frontend can suggest a specialist by prefixing the message with `[hints: ...]`. The coordinator treats hints as suggestions, not hard constraints. If the query clearly belongs to a different specialist, the coordinator routes there instead.

### Callable members factory

The coordinator uses a callable `_get_members()` factory instead of a static member list. This defers M365 agent import until runtime, avoiding MCPTools connection attempts at startup when no user token exists.

## Research Team

The Research Team coordinates two specialized agents for open-ended research tasks.

```python
from agno.agent import Agent
from agno.team import Team
from agno.team.team import TeamMode
from agno.tools.reasoning import ReasoningTools
from agno.tools.websearch import WebSearchTools

from backend.db import get_postgres_db
from backend.models import get_model

web_researcher = Agent(
    name="Web Researcher",
    role="Search the web for current information",
    model=get_model(),
    tools=[WebSearchTools(enable_search=True, enable_news=True)],
    instructions=["Search the web for relevant information.", "Cite sources with URLs."],
    markdown=True,
)

analyst = Agent(
    name="Analyst",
    role="Analyze information and synthesize findings",
    model=get_model(),
    instructions=["Analyze information from team members.", "Identify patterns and gaps."],
    markdown=True,
)

research_team = Team(
    name="Research Team",
    id="research-team",
    mode=TeamMode.coordinate,
    model=get_model(),
    db=get_postgres_db(),
    members=[web_researcher, analyst],
    tools=[ReasoningTools(add_instructions=True)],
    markdown=True,
    max_iterations=5,
    share_member_interactions=True,
    enable_agentic_memory=True,
)
```

| Member | Role | Tools |
|--------|------|-------|
| Web Researcher | Searches the web for current information | `WebSearchTools` (search + news) |
| Analyst | Analyzes and synthesizes findings | None (reasoning via instructions) |

The team leader uses `ReasoningTools` for complex multi-step delegation and `TeamMode.coordinate` to route subtasks to the appropriate member.

## Key features

| Feature | Configuration |
|---------|---------------|
| Coordination mode | `TeamMode.coordinate` routes tasks to members |
| Shared memory | `share_member_interactions=True` for cross-member context |
| History | `store_history_messages=True` persists team conversations |
| History depth | `num_history_runs=5` controls context window for team history |
| Iteration cap | `max_iterations=5` prevents runaway coordination loops |
| Reasoning | `ReasoningTools(add_instructions=True)` for the coordinator |
| Compression | `compress_tool_results=True` reduces token usage |
| Datetime context | `add_datetime_to_context=True` provides temporal awareness |
| Session memory | `db=get_postgres_db()` persists across sessions |

## Team modes

Agno supports multiple team coordination modes:

| Mode | Description |
|------|-------------|
| `TeamMode.coordinate` | Leader delegates subtasks to specific members |
| `TeamMode.collaborate` | Members work together on the same task |
| `TeamMode.route` | Leader routes the full task to one member |

See the [Agno Teams docs](https://docs.agno.com/teams/introduction) for all coordination modes.
