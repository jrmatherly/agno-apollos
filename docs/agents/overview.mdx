Apollos AI ships with six agents, two teams, and a research workflow. Each agent is a standalone Python module that exports an `Agent` instance, registered in `backend/main.py`.

## Agents

| Agent | ID | Pattern | Source |
|-------|----|---------|--------|
| Knowledge Agent | `knowledge-agent` | Agentic RAG | `backend/agents/knowledge_agent.py` |
| MCP Agent | `mcp-agent` | MCP Tool Use | `backend/agents/mcp_agent.py` |
| Web Search Agent | `web-search-agent` | Web Research | `backend/agents/web_search_agent.py` |
| Reasoning Agent | `reasoning-agent` | Chain-of-Thought | `backend/agents/reasoning_agent.py` |
| Data Analyst | `data-agent` | SQL Analysis | `backend/agents/data_agent.py` |
| M365 Agent | `m365-agent` | MCP Tool Use | `backend/agents/m365_agent.py` (opt-in) |

<Note>
The M365 Agent requires `M365_ENABLED=true` and Entra ID authentication. See [M365 configuration](/configuration/m365).
</Note>

## Teams

## How agents work

Every agent uses the [Agno](https://docs.agno.com) framework. An agent consists of:

| Component | Purpose |
|-----------|---------|
| `model` | The LLM (via shared `get_model()` factory, routed through LiteLLM Proxy) |
| `instructions` | System prompt defining behavior |
| `tools` | Functions the agent can call |
| `knowledge` | Vector database for RAG retrieval |
| `db` | PostgresDb for session memory and history |
| `pre_hooks` | Guardrails (PII detection, prompt injection) |
| `learning` | LearningMachine for agentic learning |

All agents use `get_model()` from `backend/models.py` for the model, `get_postgres_db()` for persistence, and include PII detection and prompt injection guardrails.

## Agent registration

Agents, teams, and workflows are registered in `backend/main.py`:

```python
from backend.agents.knowledge_agent import knowledge_agent
from backend.agents.mcp_agent import mcp_agent
from backend.agents.web_search_agent import web_search_agent
from backend.agents.reasoning_agent import reasoning_agent
from backend.agents.data_agent import data_agent
from backend.teams.coordinator_team import coordinator_team
from backend.teams.research_team import research_team
from backend.workflows.research_workflow import research_workflow

agent_os = AgentOS(
    name="Apollos AI",
    agents=[knowledge_agent, mcp_agent, web_search_agent, data_agent, reasoning_agent],
    teams=[coordinator_team, research_team],
    workflows=[research_workflow],
    # ...
)
```

The Coordinator Team is the primary entry point. It routes queries to the best agent based on content. Adding a new agent means creating a module and adding it to this list. See [creating agents](/agents/creating-agents).
